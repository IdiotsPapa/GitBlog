사내 Ticket 시스템

2019 — 운영개발·운영work
ASP.NET Core Oracle SAP C#

티켓을 등록하고, 장애·개선·SAP 권한 신청까지 이어지는 사내 Ticket 시스템을 구축·운영했습니다. 단순 문의 게시판이 아니라 담당자 할당, 상태 연결, 결재, 첨부·댓글, 처리 이력을 한 흐름으로 묶은 업무 플랫폼입니다.

  • 기술: C# / ASP.NET Core MVC, Oracle, SAP RFC
  • 상태: 개발 후 운영·유지보수

구성한 기능

요청자가 회사·사업장·부서·시스템·우선순위를 선택해 티켓을 등록하면, 담당자가 배정되고 상태가 바뀌며 댓글·관계자·첨부로 처리 과정이 남습니다. 유형에 따라 내부 승인·반려와 이메일 알림이 따라갑니다.

SAP 권한 요청은 역할·권한 검증과 시스템별 승인 절차를 거칩니다. SAP TO 관리는 Local TO와 Core TO로 나뉘며, 정해 둔 단계를 따라 진행 상태를 관리합니다. Excel 일괄 등록, PDF 증빙, 처리량·지연 건 대시보드도 포함합니다.

작업에서 신경 쓴 점

티켓 상태와 결재·이관 단계를 따로 두면 현재 위치를 놓치기 쉽습니다. 그래서 등록부터 종료까지 상태가 끊기지 않고 자연스럽게 이어지도록 프로세스를 구성했습니다. 화면과 알림도 같은 흐름을 따르도록 맞추고, 담당자와 요청자가 현재 티켓 상태를 바로 알 수 있게 했습니다.

처리 담당자끼리의 협업도 중요했습니다. 댓글·관계자·첨부로 진행 상황과 자료를 같은 티켓 안에서 공유할 수 있게 해, 업무가 사람마다 흩어지지 않도록 했습니다. 미처리·지연 건은 먼저 보이게 해 장애가 커지기 전에 챙길 수 있게 했습니다.
