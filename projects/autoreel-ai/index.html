AutoReel AI — 숏폼 영상 생성 파이프라인

2026.03 — 04설계·구현ai
Python AI Agent Claude Automation

시작한 이유

트렌드 하나를 골라 숏폼 영상으로 올리기까지는 생각보다 손이 많이 갑니다. 자료를 모으고, 대본을 쓰고, 음성과 영상을 맞추고, 자막을 입힌 다음 게시까지 해야 합니다. 이 흐름을 어디까지 자동화할 수 있는지 확인하려고 Python으로 MVP를 만들었습니다.

  • 작업 기간: 2026년 3월 19일 ~ 4월 19일
  • 상태: 핵심 흐름이 동작하는 MVP, 운영 제품으로는 보완 중
  • 역할: 파이프라인 설계·구현·실행 검증

가장 오래 붙잡았던 문제

처음에는 각 API를 호출해 결과를 받으면 끝날 거라 생각했습니다. 실제로 연결해 보니 AI·TTS·영상 API 중 하나만 실패해도 앞에서 만든 결과까지 다시 처리해야 하는 문제가 더 컸습니다. 비용이 드는 호출을 무작정 반복할 수도 없었습니다.

그래서 작업을 한 덩어리로 실행하지 않고 아래 단계로 나눴습니다.

트렌드 수집
  → 유사 키워드 병합·점수화
  → AI 대본 생성
  → TTS 음성 생성
  → 영상 소스 확보
  → 자막·세로 영상 합성
  → 형식 QC
  → 사람 검수 또는 자동 승인
  → YouTube 게시

각 단계의 상태와 산출물 이력은 로컬 JSON에 남겼고, 일부 외부 API가 실패해도 다음 동작을 확인할 수 있도록 단계별 fallback을 뒀습니다. 반복 실행과 여러 대본의 일괄 처리도 이 구조 위에 붙였습니다.

영상 한 편이 만들어지는 과정

여러 출처에서 트렌드를 병렬로 모은 뒤 비슷한 키워드를 합치고 가중치로 순서를 정합니다. 선택한 주제는 Claude나 OpenAI로 보내 [Hook] / [Body] / [CTA] 구조의 대본으로 만들었습니다. 점수가 낮을 때는 횟수를 제한해 다시 쓰게 했고, AI 대신 직접 작성한 파일이나 문자열을 넣는 경로도 열어 뒀습니다.

한국어 음성은 ElevenLabs TTS로 만들고, Pexels·Pixabay에서 스톡 영상을 찾거나 생성형 영상 API를 연결해 보는 실험을 했습니다. 이후 FFmpeg로 1080×1920 세로 영상을 합성합니다. 자막은 TTS 길이에 맞춰 SRT로 만들되, 긴 문장이 모바일 화면을 가득 채우지 않도록 문장과 글자 수 기준으로 나눴습니다.

마지막에는 길이·해상도·금지어를 확인합니다. 통과한 결과는 사람이 승인하거나 점수 기준으로 자동 승인한 뒤 Google OAuth와 YouTube Data API를 통해 업로드할 수 있습니다.

자동 QC를 만들고도 수동 검수를 남긴 이유

현재 QC가 확인하는 것은 형식입니다. 영상 길이와 해상도가 맞는지, 금지어가 들어갔는지는 코드로 판정할 수 있지만 내용이 사실인지, 영상과 대본이 의미상 맞는지, 저작권 문제가 없는지는 판단하지 못합니다.

자동 승인은 처리 속도를 확인하기 위한 선택지일 뿐 품질 보장이 아닙니다. 그래서 수동 승인 경로를 함께 두었습니다. AI도 설계 초안과 반복 구현에 활용했지만, 생성된 코드를 그대로 믿기보다는 실행 로그와 실제 산출물을 확인했습니다.

사용한 도구

Python과 PowerShell로 실행 흐름을 구성했고 APScheduler로 반복 작업을 다뤘습니다. 대본 생성에는 Anthropic Claude와 OpenAI, 음성에는 ElevenLabs, 영상 합성에는 FFmpeg를 사용했습니다. 영상 소스는 Pexels·Pixabay·생성형 영상 API를 연결했고, 게시에는 YouTube Data API와 Google OAuth를 사용했습니다.

아직 남은 일

사실 검증과 출처 인용, 의미 기반 영상 품질 판정은 아직 구현하지 못했습니다. Instagram·TikTok 게시도 지원하지 않습니다. 자동 테스트와 큐, 데이터베이스, 충분한 재시도 정책이 부족하고 의존성 버전 고정과 배포 재현성도 더 보완해야 합니다.

따라서 지금 단계는 완전 자동 콘텐츠 서비스가 아니라, AI 결과물을 검수 가능한 단계로 이어 본 자동화 MVP입니다.

공개 시 주의할 점

API 키와 OAuth 토큰은 콘텐츠와 분리해야 합니다. 공개 저장소로 전환하기 전에는 Git 이력까지 확인하고, 노출 가능성이 있는 키는 폐기·교체해야 합니다. 인증 파일, 실행 로그와 미공개 산출물도 공개하지 않습니다.
