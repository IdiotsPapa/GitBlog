AutoReel AI — 숏폼 영상 생성 파이프라인

2026.03 — 04설계·구현ai
Python AI Agent Claude Automation

트렌드 하나를 골라 숏폼으로 올리기까지 손이 많이 갑니다. 자료를 모으고 대본·음성·영상·자막·게시까지 이어지는 흐름을 어디까지 자동화할 수 있는지 확인하려고 Python MVP를 만들었습니다.

  • 작업 기간: 2026년 3월 19일 ~ 4월 19일
  • 상태: 핵심 흐름이 동작하는 MVP (운영 제품으로는 보완 중)
  • 역할: 파이프라인 설계·구현·실행 검증

파이프라인 구조

각 API를 한 번에 이어 붙이면 한 단계만 실패해도 앞 결과를 다시 만들게 됩니다. 그래서 단계를 나눴습니다.

트렌드 수집 → 키워드 병합·점수화 → AI 대본
  → TTS → 영상 소스 → 자막·세로 합성 → 형식 QC
  → 사람 검수 또는 자동 승인 → YouTube 게시

단계별 상태·산출물은 로컬 JSON에 남기고, 일부 API 실패 시에도 다음 동작을 확인할 수 있도록 fallback을 뒀습니다.

영상 한 편이 만들어지는 과정

여러 출처에서 트렌드를 병렬로 모은 뒤 유사 키워드를 합치고 가중치로 순서를 정합니다. 대본은 Claude·OpenAI로 [Hook] / [Body] / [CTA] 구조를 만들고, 점수가 낮으면 횟수를 제한해 다시 쓰게 했습니다. 직접 작성한 대본을 넣는 경로도 있습니다.

음성은 ElevenLabs TTS, 영상은 Pexels·Pixabay·생성형 API를 연결한 뒤 FFmpeg로 1080×1920 세로 영상을 합성합니다. 자막은 TTS 길이에 맞춰 SRT로 나누고, QC에서는 길이·해상도·금지어를 확인합니다. 통과분은 수동 승인 또는 점수 기준 자동 승인 후 YouTube Data API로 업로드할 수 있습니다.

자동 QC만으로 끝내면 안 되는 이유

현재 QC는 형식만 봅니다. 내용의 사실·의미 일치·저작권은 판정하지 않습니다. 자동 승인은 속도 확인용이며, 수동 승인 경로를 함께 둡니다.

아직 남은 일

사실 검증·출처 인용·의미 기반 품질 판정, Instagram·TikTok 게시, 자동 테스트·큐·DB·재시도 정책은 미구현입니다. 완전 자동 서비스가 아니라 검수 가능한 단계로 이어 본 자동화 MVP입니다.

API 키·OAuth 토큰·실행 로그·미공개 산출물은 공개하지 않습니다.
