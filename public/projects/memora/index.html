Memora — AI 학습 카드 앱

2026.05 — 07개발mobile
React Native TypeScript Expo Claude

카드를 만드는 것보다 어려웠던 일

Memora는 필기 사진·PDF·텍스트를 플래시카드와 퀴즈로 바꾸고 복습까지 이어 주는 모바일 앱입니다.

  • 작업 기간: 2026년 5월 15일 ~ 7월 10일
  • 상태: 핵심 학습 흐름을 구현한 베타 수준, 일부 실험 기능 비활성
  • 플랫폼: Expo / React Native

Claude로 카드 몇 장을 생성하는 것 자체는 어렵지 않았습니다. 막상 써 보니 더 중요한 것은 “이 답이 원래 자료의 어디에서 나왔는가”였습니다. 생성 결과만 따로 쌓이면 틀린 내용을 발견해도 확인할 길이 없었습니다. 그래서 카드에 원본 이미지 인덱스와 위치 정보를 함께 저장하고, 학습 중 바로 노트로 돌아갈 수 있게 했습니다.

결과적으로 앱의 흐름은 다음처럼 잡혔습니다.

학습 자료 등록
  → AI 카드·퀴즈 생성
  → 카드 피드·퀴즈 학습
  → 결과와 약점 확인
  → 다음 복습 일정 계산

자료를 넣는 과정에서 생긴 문제

텍스트뿐 아니라 노트 이미지와 PDF도 저장하고 분석할 수 있게 했습니다. 사진은 Image Picker, 문서는 Document Picker와 File System을 사용했습니다.

PDF를 처음 다룰 때는 JSON에 base64로 넣어 보내는 방식이 간단해 보였습니다. 하지만 파일이 커지면 메모리 사용량과 요청 크기가 함께 커졌습니다. 이후 Anthropic Claude Files API로 파일을 먼저 업로드하도록 바꾸고, 실행 환경에 따라 네이티브 multipart와 FormData fallback을 구분했습니다.

생성된 덱·카드·퀴즈는 중간에 일부만 저장되지 않도록 트랜잭션으로 한꺼번에 기록합니다. Android에서 SQLite에 undefined가 바인딩될 때 생길 수 있는 예외도 있어서, DB에 넣기 전에 값을 정규화했습니다. 스키마를 늘릴 때는 필요한 컬럼이 있는지 확인한 뒤 추가하도록 처리했습니다.

학습 기록은 오프라인에 남겼다

학습 자료·덱·카드·퀴즈·학습 로그·계획은 Expo SQLite에 저장합니다. 사진과 PDF는 로컬 파일 경로를 연결했습니다. AI 생성이 필요한 순간에만 외부 API를 호출하고, 이미 만든 카드와 학습 기록은 오프라인에서도 볼 수 있는 구조를 목표로 했습니다.

카드는 뒤집은 뒤 알고 있음/모름을 기록할 수 있고, 퀴즈는 객관식·빈칸·O/X·직접 입력 방식을 지원합니다. 결과를 바탕으로 일별 학습량과 약한 카드를 집계하며, SM-2 방식으로 다음 복습 일정을 계산합니다. 덱·기간·요일별 학습 계획과 복습 알림도 연결했습니다.

학습을 계속할 동기를 주기 위해 XP·레벨·스트릭·일일 목표를 넣었고, 화면은 휴대폰과 태블릿 크기에 대응하도록 구성했습니다. 과목이나 시험 단위로 학습 트랙을 나눌 수도 있습니다.

만들었지만 켜지 않은 기능

Three.js로 3D 학습 방과 꾸미기 기능도 구현했습니다. 다만 핵심 카드 학습과 비교해 안정화가 충분하지 않았고, 기능 플래그로 릴리스 범위에서 제외했습니다. 구현한 시간이 아깝더라도 불안정한 기능을 베타에 같이 내놓는 것은 맞지 않다고 판단했습니다.

기술 구성과 남은 검증

React Native, React, TypeScript와 Expo SDK를 사용했고 화면 이동은 React Navigation으로 구성했습니다. 알림은 Expo Notifications, AI 분석은 Anthropic Claude Messages·Files API를 사용합니다. 카드·퀴즈·DB 핵심 로직에는 Jest와 jest-expo로 단위 테스트를 작성했습니다.

자동 E2E와 스토어 배포 검증은 아직 남아 있습니다. 무엇보다 클라이언트에 AI API 키를 넣는 현재 구조는 프로덕션에 적합하지 않습니다. 실제 서비스로 옮기려면 백엔드 프록시와 인증, 호출량 제한, 키 보호가 필요합니다.

따라서 Memora는 정식 출시 제품이 아니라 핵심 학습 경험을 검증한 후기 MVP입니다.
