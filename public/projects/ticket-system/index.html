사내 Ticket 시스템

2019 — 운영개발·운영work
ASP.NET Core Oracle SAP C#

IT 지원 요청을 등록하고, 승인·처리·SAP 변경 이관까지 이어지는 사내 Ticket 시스템을 구축·운영했습니다. 단순 문의 게시판이 아니라 담당 배정, 상태 전이, 결재, 첨부·댓글, 처리 이력을 한 흐름으로 묶은 업무 플랫폼입니다.

  • 기술: C# / ASP.NET Core MVC, Oracle, SAP RFC
  • 상태: 개발 후 운영·유지보수

구성한 기능

요청자가 회사·사업장·부서·시스템·우선순위를 선택해 티켓을 등록하면, 담당자가 배정되고 상태가 바뀌며 댓글·관계자·첨부로 처리 과정이 남습니다. 유형에 따라 내부 승인·반려와 이메일 알림이 따라갑니다.

SAP 연계 요청은 역할·권한 검증과 시스템별 승인 절차를 거칩니다. 변경 이관은 Local TO와 Core TO로 나뉘며, 화면 코드에 명시된 정상 전이 단계를 따라 진행 상태를 관리합니다. Excel 일괄 등록, PDF 증빙, 처리량·지연 건 대시보드도 포함합니다.

작업에서 신경 쓴 점

티켓 상태와 결재·이관 단계를 따로 두면 현재 위치를 놓치기 쉽습니다. 등록부터 처리 완료까지 화면과 알림이 같은 업무 흐름을 따르도록 맞췄고, 미처리·지연 건을 먼저 보이게 해 장애가 커지기 전에 챙길 수 있게 했습니다.

내부 시스템명·서버·계정·DB 식별자는 보안상 공개하지 않습니다.
