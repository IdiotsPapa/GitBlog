상담 매뉴얼이 문서로만 있으면 다음 단계를 매번 찾아야 합니다. 관리자는 그래프로 절차를 편집하고, 상담사는 한 단계씩 따라가는 형태로 만들어 봤습니다.

- 작업 기간: 2026년 4월 24일 ~ 5월 8일
- 상태: 웹 MVP + 오프라인 모바일 프로토타입 (운영 시스템 아님)

## 웹에서 만든 흐름

React Flow로 절차를 그립니다. 상담사 화면은 현재 노드·스크립트·체크리스트를 보여 주고 분기로 이동합니다. 방문 단계 되돌리기·메모·세션 초기화도 있습니다.

관리자는 노드 CRUD와 분기 연결, JSON 가져오기/내보내기를 할 수 있습니다. 저장 전 루트·연결 대상 존재 여부를 검사합니다. 순환 그래프·도달 불가 노드·삭제 후 잔여 참조 검사는 아직 없습니다.

저장은 Express + JSON 파일이며, 임시 파일 작성 후 교체합니다. 화면은 API 계약만 바라보게 해 저장소 교체 영향을 줄이려 했습니다. PostgreSQL은 설계 단계입니다.

## 모바일

Expo / React Native로 관리자·상담사 화면을 구성하고 AsyncStorage에 둡니다. 네트워크 없이 동작하는 쪽을 목표로 했으나, 웹과의 버전·동기화·충돌 정책은 미구현입니다.

## MVP에서 멈춘 부분

인증·권한, 실시간 배포·롤백, 이력·통계, 자동화 테스트는 없습니다. 확인한 것은 **노드 그래프로 상담 흐름을 따라가고 편집하는 경험이 성립하는지**입니다.
