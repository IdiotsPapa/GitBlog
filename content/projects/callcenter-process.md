## 프로젝트 개요

상담사가 정해진 절차를 따라 스크립트·체크리스트·분기를 확인하고, 관리자는 같은 프로세스를 그래프로 편집하는 시스템입니다.

- 작업 기간: 2026년 4월 24일 ~ 5월 8일 중심
- 상태: 웹 MVP + 오프라인 모바일 프로토타입

## 문제 정의

상담 매뉴얼이 문서로만 존재하면, 상담사는 현재 상황에서 어느 단계로 가야 하는지 직접 찾아야 합니다. 이 프로젝트는 상담 흐름을 **노드와 분기 관계**로 표현해 다음 행동을 안내하는 것을 목표로 했습니다.

```text
관리자: 노드·분기 편집 → 검증 → 저장
                         ↓
상담사: 현재 단계 → 스크립트·체크리스트 → 다음 분기
```

## 웹에서 구현한 기능

- React Flow 기반 상담 절차 시각화
- 현재 노드 강조와 분기 이동
- 방문 단계로 되돌아가기
- 단계별 상담 스크립트·체크리스트
- 상담 메모와 세션 초기화
- 관리자 노드 추가·수정·삭제
- 분기 연결과 JSON 가져오기·내보내기
- 저장 전 루트·연결 대상 검증
- Express 기반 파일 저장 API
- 임시 파일 작성 후 교체하는 원자적 JSON 저장

## 모바일에서 실험한 기능

- Expo / React Native 기반 관리자·상담사 화면
- 기기 내부 AsyncStorage 저장
- 노드 CRUD와 분기 대상 검색
- 휴대폰·태블릿 레이아웃
- 단계 이동·방문 경로·메모·진행률

모바일 구현은 아직 진행 중이며 릴리스된 기능으로 보지 않습니다.

## 기술 스택

- React 18 / Vite / Tailwind CSS
- React Flow
- Node.js / Express
- Expo / React Native / TypeScript
- React Navigation / AsyncStorage

## 기술적 고민

### 그래프가 깨지지 않도록 검증하기

분기 대상 노드가 삭제되거나 잘못된 ID를 참조하면 상담이 중간에서 멈춥니다. 저장 전에 루트와 연결 대상이 실제로 존재하는지 확인했습니다.

앞으로는 순환 그래프, 도달할 수 없는 노드, 삭제 후 남은 참조까지 검증 범위를 넓혀야 합니다.

### 파일 저장에서 DB 전환을 고려하기

MVP는 JSON 파일을 사용하지만, 클라이언트는 API 계약만 바라보게 했습니다. 저장소를 데이터베이스로 교체하더라도 화면 변경을 줄이기 위한 선택입니다.

### 웹 중심과 오프라인 모바일의 차이

웹은 중앙 서버의 최신 프로세스를 사용하고, 모바일은 네트워크 없이 동작할 수 있어야 합니다. 두 모델을 함께 가져가려면 버전·동기화·충돌 정책이 필요합니다.

## 현재 한계

- 인증·권한 분리 미구현
- 실시간 배포와 버전 롤백 미구현
- 상담 이력·통계·히트맵 미구현
- PostgreSQL은 설계 단계
- 자동화 테스트 없음
- 진행률 계산 방식 개선 필요

따라서 운영 시스템이 아니라, **노드 그래프 기반 상담 흐름과 관리자 편집 경험을 검증한 프로토타입**입니다.
