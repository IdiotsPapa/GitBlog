상담 매뉴얼이 문서로만 있으면 상담사는 매번 현재 상황에 맞는 다음 단계를 직접 찾아야 합니다. 이 흐름을 노드와 분기로 바꾸면 어떨지 확인해 보고 싶어, 관리자는 절차를 그래프로 편집하고 상담사는 한 단계씩 따라가는 형태로 만들었습니다.

작업은 2026년 4월 24일부터 5월 8일까지 집중해서 진행했습니다. 결과물은 운영 시스템이 아니라 **웹 MVP와 오프라인 모바일 프로토타입**입니다.

## 먼저 웹에서 흐름을 만들었다

웹에서는 React Flow로 상담 절차를 그렸습니다. 상담사 화면은 현재 노드를 강조하고, 단계별 스크립트와 체크리스트를 보여 준 뒤 선택한 분기로 이동합니다. 이미 방문한 단계로 돌아가거나 메모를 남기고 세션을 초기화하는 기능도 넣었습니다.

관리자 화면에서는 노드를 추가·수정·삭제하고 분기를 연결할 수 있습니다. 절차는 JSON으로 가져오거나 내보낼 수 있고, 저장 전에는 루트와 연결 대상이 실제로 존재하는지 확인합니다. 분기 대상이 삭제됐거나 잘못된 ID를 가리키면 상담이 중간에서 멈추기 때문입니다. 다만 순환 그래프, 도달할 수 없는 노드, 삭제 후 남은 참조까지는 아직 검사하지 못합니다.

저장은 Express API와 JSON 파일로 처리했습니다. 파일을 바로 덮어쓰지 않고 임시 파일을 작성한 뒤 교체하는 방식으로 구성했습니다. MVP에서는 파일이면 충분했지만, 화면은 API 계약만 바라보게 해 나중에 저장소를 데이터베이스로 바꿀 때 영향을 줄이려 했습니다. PostgreSQL은 아직 설계 단계입니다.

## 모바일은 오프라인 가능성을 시험했다

Expo와 React Native로 관리자·상담사 화면을 별도로 실험했습니다. AsyncStorage에 데이터를 보관하고, 노드 CRUD와 분기 대상 검색, 단계 이동, 방문 경로, 메모, 진행률을 구현했습니다. 휴대폰과 태블릿 레이아웃도 나눴습니다.

웹이 중앙 서버의 최신 절차를 보는 방식이라면 모바일은 네트워크 없이도 동작해야 합니다. 둘을 함께 운영하려면 버전, 동기화, 충돌 정책이 필요하지만 여기까지는 구현하지 않았습니다. 모바일 역시 아직 진행 중인 프로토타입이며 릴리스된 기능은 아닙니다.

사용한 기술은 웹의 React 18, Vite, Tailwind CSS, React Flow와 Node.js, Express, 모바일의 Expo, React Native, TypeScript, React Navigation, AsyncStorage입니다.

## MVP에서 멈춘 부분

인증과 권한 분리, 실시간 배포와 버전 롤백, 상담 이력·통계·히트맵은 구현하지 않았습니다. 자동화 테스트도 없고 진행률 계산 방식은 개선이 필요합니다.

이 작업에서 확인한 것은 완성된 콜센터 운영 환경이 아니라, **노드 그래프로 상담 흐름을 따라가고 관리자가 같은 흐름을 편집하는 경험이 성립하는지**였습니다.
