## 프로젝트 개요

학생의 필기 사진·PDF·텍스트를 AI로 분석해 플래시카드와 퀴즈로 만들고, 반복 학습을 돕는 모바일 앱입니다.

- 작업 기간: 2026년 5월 15일 ~ 7월 10일
- 상태: 핵심 학습 흐름을 구현한 베타 수준, 일부 실험 기능 비활성
- 플랫폼: Expo / React Native

## 핵심 사용자 흐름

```text
학습 자료 등록
  → AI 카드·퀴즈 생성
  → 카드 피드·퀴즈 학습
  → 결과와 약점 확인
  → 다음 복습 일정 계산
```

앱의 중심은 화려한 AI 화면이 아니라, 생성된 학습 자료가 **원본 노트와 연결되고 다시 복습되는 과정**입니다.

## 구현한 기능

- 텍스트·노트 이미지·PDF 자료 저장
- 과목·시험 등 학습 트랙 분류
- Claude 기반 카드·퀴즈 생성
- 이미지와 PDF 분석
- 생성 결과를 덱·카드·퀴즈로 일괄 저장
- 카드 뒤집기와 알고 있음/모름 처리
- 객관식·빈칸·O/X·직접 입력 퀴즈
- 카드에서 원본 노트 위치 확인
- SM-2 방식 복습 일정 계산
- 학습 로그·일별 학습량·약한 카드 집계
- 덱·기간·요일 기반 학습 계획
- 복습 알림
- XP·레벨·스트릭·일일 목표
- 휴대폰·태블릿 반응형 화면

## 데이터 설계

학습 자료·덱·카드·퀴즈·학습 로그·계획을 로컬 SQLite에 저장합니다. 일괄 카드·계획 저장에는 트랜잭션을 사용하고, 사진과 PDF는 로컬 파일 경로를 연결합니다.

오프라인에서 학습 기록을 사용할 수 있도록 하면서, AI 생성이 필요한 순간에만 외부 API를 호출하는 구조를 지향했습니다.

## 개발하며 해결한 문제

### PDF 전송 방식

대용량 PDF를 JSON 안의 base64로 보내면 메모리와 요청 크기 문제가 생깁니다. 파일 업로드 API를 사용하고, 환경에 따라 네이티브 multipart와 FormData fallback을 구분했습니다.

### AI 결과와 원본의 연결

생성된 카드만 남으면 사용자가 근거를 확인하기 어렵습니다. 카드에 원본 이미지 인덱스와 위치 정보를 연결해 노트로 되돌아갈 수 있게 했습니다.

### SQLite 바인딩 안정성

Android에서 `undefined` 값 바인딩으로 발생할 수 있는 예외를 막기 위해 입력값을 정규화했습니다. 스키마 확장은 필요한 컬럼을 확인해 추가하는 방식으로 처리했습니다.

### 실험 기능 격리

Three.js 기반 3D 학습 방과 꾸미기 기능을 구현했지만 안정화가 충분하지 않아 기능 플래그로 릴리스 범위에서 제외했습니다. 구현했다는 이유만으로 사용자에게 노출하기보다, 핵심 학습 흐름의 안정성을 우선했습니다.

## 테스트와 현재 상태

카드·퀴즈·DB 등 핵심 로직을 대상으로 단위 테스트를 작성했습니다. 다만 자동 E2E와 스토어 배포 검증은 남아 있습니다.

또한 클라이언트에 AI API 키를 넣는 현재 구조는 프로덕션에 적합하지 않습니다. 실제 서비스로 전환하려면 백엔드 프록시, 인증, 호출량 제한과 키 보호가 필요합니다.

이 프로젝트는 정식 출시 제품이 아니라 **핵심 학습 경험을 검증한 후기 MVP**입니다.
