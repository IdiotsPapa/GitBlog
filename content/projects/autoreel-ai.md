## 프로젝트 개요

외부 트렌드를 수집하고 AI로 대본을 작성한 뒤, 음성·영상·자막을 합성해 숏폼 영상으로 만드는 자동화 MVP입니다.

- 작업 기간: 2026년 3월 19일 ~ 4월 19일
- 상태: 핵심 흐름이 동작하는 MVP, 운영 제품으로는 보완 중
- 역할: 파이프라인 설계·구현·실행 검증

## 전체 흐름

```text
트렌드 수집
  → 유사 키워드 병합·점수화
  → AI 대본 생성
  → TTS 음성 생성
  → 영상 소스 확보
  → 자막·세로 영상 합성
  → 형식 QC
  → 사람 검수 또는 자동 승인
  → YouTube 게시
```

단일 API 호출 예제가 아니라, 단계별 상태와 실패 처리까지 포함한 **작업 파이프라인**을 만드는 데 초점을 맞췄습니다.

## 실제 구현한 기능

- 여러 출처의 트렌드 병렬 수집
- 유사 키워드 병합 및 가중치 기반 점수화
- Claude·OpenAI를 이용한 `[Hook] / [Body] / [CTA]` 대본 생성
- 품질 점수가 낮은 대본의 제한적 재작성
- 직접 작성한 파일·문자열 대본 입력
- TTS 기반 한국어 음성 생성
- 스톡 영상 검색 및 생성형 영상 연동 실험
- 문장 길이에 맞춘 SRT 자막 생성
- FFmpeg 기반 1080×1920 영상 합성
- 길이·해상도·금지어를 확인하는 형식 QC
- JSON 기반 작업 상태·산출물 이력 관리
- 수동 승인과 점수 기반 자동 승인
- OAuth 기반 YouTube 업로드
- 반복 실행과 여러 대본의 일괄 처리

## 설계하며 고민한 점

### 외부 API 실패가 전체 작업 실패가 되지 않도록

AI·TTS·영상 API는 비용과 장애 가능성이 있습니다. 단계별 fallback을 두어 일부 API가 실패하더라도 파이프라인 구조와 다음 단계를 검증할 수 있게 했습니다.

### 생성 속도와 사람 검수의 균형

자동 승인은 빠르지만 현재 QC는 길이·해상도 같은 **형식적 품질**만 판단합니다. 사실성, 저작권, 대본과 영상의 의미적 일치는 사람이 확인해야 합니다. 따라서 수동 검수와 자동 승인을 모두 지원하되, 자동화가 곧 품질 보장을 의미하지 않도록 구분했습니다.

### 모바일 가독성

TTS 길이에 맞춰 영상과 자막 시간을 조정하고, 긴 문장은 문장·글자 수 기준으로 나눴습니다. 생성 성공 여부뿐 아니라 세로 화면에서 읽히는지도 고려했습니다.

## AI 도구를 다룬 방식

AI는 설계 초안과 반복 구현을 빠르게 만드는 데 활용했습니다. 생성된 결과는 그대로 신뢰하지 않고 실행 로그와 산출물을 확인했습니다.

특히 API 키와 OAuth 토큰은 콘텐츠와 분리해야 합니다. 공개 저장소 전환 전에는 Git 이력까지 점검하고, 노출 가능성이 있는 키를 폐기·교체해야 한다는 점도 확인했습니다.
