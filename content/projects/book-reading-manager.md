## 프로젝트 개요

부모가 자녀별로 책, 독서 완료 기록과 독서 계획을 관리하는 Flutter 앱입니다.

- 초기 개발: 2025년 10월 15일 ~ 11월 2일
- 후속 보강: 2026년 3월
- 상태: 핵심 CRUD와 CSV 가져오기를 구현한 초기 MVP

## 구현한 기능

- 자녀 프로필 등록·수정·삭제
- 책 등록·수정·삭제
- 책에 자녀·읽은 날짜·태그·메모 연결
- 자녀별 책 목록 필터
- 독서 계획과 진행률 관리
- 자녀별 독서 기록과 월별 권수 집계
- 가족 전체·자녀별 홈 요약
- CSV 기반 도서 대량 가져오기

## 기술 스택

- Flutter / Dart
- Hive / Hive Flutter
- Hive Generator / build_runner
- file_picker / csv
- intl / path_provider

## 로컬 데이터 설계

책·자녀·계획을 각각 Hive 박스에 저장하고 문자열 ID로 관계를 연결했습니다.

관계형 DB처럼 외래 키와 자동 cascade가 없기 때문에, 자녀를 삭제할 때 연결된 책의 배정을 해제하고 해당 자녀의 계획을 코드에서 직접 정리했습니다.

## CSV 가져오기

여러 권의 책을 한 번에 등록하기 위해 CSV 업로드를 구현했습니다.

- 필수 헤더와 BOM 처리
- 날짜 형식 검증
- 자녀 이름 매핑
- 제목·저자·자녀 기준 중복 확인
- 중복 시 새 행 대신 기존 책 갱신
- 잘못된 행은 전체 실패가 아니라 개별 경고

단순 파일 읽기보다, 사용자가 일부 잘못된 데이터를 넣더라도 **정상 행까지 잃지 않게 하는 것**을 중요하게 봤습니다.

## 개발하며 고민한 점

### 로컬 모델 확장

Hive 모델은 필드 번호가 저장 데이터와 연결되므로 기존 번호를 유지하며 확장해야 합니다. 스키마 변경이 기존 로컬 데이터를 깨뜨리지 않도록 주의했습니다.

### UI 생명주기

BottomSheet가 닫히는 애니메이션 중 컨트롤러를 너무 일찍 해제하면 예외가 발생할 수 있습니다. 화면 전환과 컨트롤러 생명주기를 함께 확인했습니다.

### 한글 표시

자녀 이름의 첫 글자를 안전하게 표시하기 위해 코드 유닛 대신 grapheme cluster를 다루는 패키지를 사용했습니다.

## 구현하지 않은 항목

화면에 아이디어로 남아 있던 OCR, Google Books 연동, AI 추천, 백업·복원은 구현되지 않았습니다. 독서 계획의 진행률도 독서 기록과 자동 연동되지 않고 사용자가 조절합니다.

따라서 “AI 독서 앱”이 아니라, **자녀별 로컬 독서 기록과 계획을 관리하는 Flutter MVP**로 소개합니다.
