## 프로젝트 개요

카드 결제 알림을 자동으로 거래 내역으로 변환하고, 개인 지출·그룹 정산·월별 가계부를 관리하는 Flutter 앱입니다.

- 작업 기간: 2026년 5월 27일 ~ 6월 11일
- 상태: Android 중심의 기능성 MVP
- 핵심: Flutter 앱과 Android 시스템 알림의 네이티브 연동

## 구현한 기능

- 홈·정산·가계부·설정 4개 탭
- 거래 수동 추가
- 개인/그룹 거래와 지출자·결제자 지정
- 월별 거래·총지출·받을 돈 표시
- 멤버·그룹 관리
- 거래에 사용 중인 멤버·그룹 삭제 방지
- 멤버별 잔액과 송금 방향 계산
- 월별·카테고리별 소비 집계
- Android 결제 알림 자동 수집
- 승인 취소·거절 알림 제외
- 금액·가맹점·카드사 추출
- 일정 시간 내 중복 거래 방지
- 알림 접근 권한 확인과 설정 화면 이동

## 기술 구성

- Flutter / Dart
- Provider / ChangeNotifier
- SharedPreferences
- Android Kotlin
- MethodChannel / EventChannel
- NotificationListenerService

## 앱이 꺼져 있을 때의 알림 처리

결제 알림은 Flutter 화면이 실행 중일 때만 발생하지 않습니다. 앱이 종료된 상태에서도 Android 서비스가 알림을 받을 수 있어야 했습니다.

```text
Android NotificationListenerService
  ├─ 앱 실행 중: EventChannel로 Flutter에 전달
  └─ 앱 미실행: 네이티브 SharedPreferences 큐에 저장
                     ↓
                다음 앱 실행 시 읽고 거래로 변환
```

Flutter와 Android 네이티브 사이의 생명주기가 다르기 때문에, 이벤트 전달과 임시 저장을 분리했습니다.

## 정산 방식

각 거래에 지출자와 실제 결제자를 기록해 멤버별 채권·채무를 계산합니다. 이후 채권자와 채무자의 잔액을 비교해 필요한 송금 목록으로 단순화합니다.

현재는 한 거래를 여러 사람이 비율 또는 균등 분할하는 구조가 아니라, **단일 지출자·단일 결제자 기준**입니다.

## 개발하며 고민한 점

- 카드사마다 다른 알림 문구를 어떻게 정규화할지
- 승인 취소와 결제 거절을 거래에서 제외하는 방법
- 앱 비실행 중 알림을 유실하지 않는 방법
- 같은 거래가 반복 수신될 때 중복을 막는 기준
- 거래가 참조 중인 멤버·그룹의 삭제를 막아 데이터 일관성을 지키는 방법

## 현재 한계

- Android만 결제 알림 자동 수집 지원
- 카드사별 실제 문구 변화에 취약한 휴리스틱 파서
- 거래 수정·삭제 UI 보완 필요
- 로그인·백업·동기화 없음
- 정식 배포와 폭넓은 실기기 검증 필요
- 데이터가 늘면 SharedPreferences 대신 로컬 DB 전환 필요

완성된 서비스로 포장하기보다, **Flutter와 Android 네이티브 기능을 연결해 핵심 가설을 검증한 MVP**로 기록합니다.
