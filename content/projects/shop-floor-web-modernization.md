생산현장 작업자가 작업지시·진행·실적·비가동·불량을 처리하던 단말 시스템을 유지보수하고, 웹 전환 가능성을 검토한 작업입니다.

- 설치형 유지보수: 2019년 ~ 2023년 확인 가능
- 웹 전환 PoC: 2026년 7월
- 상태: 설치형은 운영·유지보수, 웹 전환본은 초기 PoC

## 기존 시스템

현장 PC에 설치하는 Windows 데스크톱 애플리케이션입니다. 작업장·라인·근무조를 고른 뒤 작업지시와 계획수량을 확인하고, 시작·종료·실적·불량·재투입·비가동을 처리합니다. 바코드·포장·라벨과 현장별 예외, 외부 연계도 포함돼 있었습니다.

기능이 쌓이면서 오래된 UI·플랫폼 의존성과 PC별 설치·업데이트 부담이 커졌습니다. 레거시 유지보수 경험을 바탕으로 .NET 8·React로 브라우저 기반 구조와 일부 기능을 만들었습니다.

## 한 번에 바꾸지 않은 이유

화면만 웹으로 옮기면 기술만 바뀌고 복잡성은 남습니다. 상태가 바뀌는 업무 규칙과 현장 예외를 먼저 파악한 뒤, 조회·공통 기능부터 옮기는 점진적 방식을 택했습니다.

PoC에서는 REST API 골격과 키오스크 UI, 로컬 데이터 재현 환경을 마련했습니다. 인증·전체 업무·장비 연동·E2E 검증까지는 끝내지 않았습니다.

## 현장 시스템이라서 남은 고민

브라우저 화면은 서버 연결에 더 의존하므로 연결 표시·재시도·중복 요청 방지, 로컬·서버 상태 일관성을 같이 봐야 합니다. 바코드·라벨처럼 로컬 장비 기능은 일반 웹 API만으로 옮기기 어려워, 브라우저 지원·로컬 브리지·에이전트 중 적절한 방식을 기능별로 판단해야 합니다.

장기간 운영된 시스템에는 화면만으로는 알기 어려운 예외가 많습니다. 작은 범위부터 옮기고 기존 결과와 비교하는 것이 회귀 위험을 줄이는 현실적인 방법이었습니다.

회사·고객·공장·장비·DB 식별 정보와 실제 화면은 보안상 공개하지 않습니다.
